# Роботизированный комплекс санитарной обработки, сбора и передачи агротехнических данных садовых насаждений.

В помощь садоводам и огородникам больших хозяйств создано немало автоматизированных машин и устройств, для облегчения тяжелого  труда. На сегодняшний день это дорогостоящие комплексы, даже их аренда.  Чем меньше хозяйство, тем меньше возможностей в найме специалистов. А если учесть садоводов-дачников и  садоводов-любителей частных домов, то здесь приходится обходиться собственными силами и опытом. 

Данное устройство – это профессиональный помощник садоводу в уходе за растениями.

Идея создать эту установку возникла в связи со сложностью обработки моего собственного сада. Дело в том, что для обработки нужно подходящее время, чтобы не сделать ожоги на листьях и не потравить пчёл (полезных насекомых), поэтому обработку надо проводить с наступлением сумерек и на рассвете.  

Проблема - подобрать время, когда минимальный ветер и нет дождя. Эти 3 показателя достаточно сложно собрать вместе. А бывает, что всё совпало, но садовод занят, там работа, гости и тд.  К тому же, ночью с опрыскивателем садоводу сложно передвигаться по саду. А данная система сама выберет подходящие условия. Измерит температуру, влажность, скорость ветра. Затем если всё совпало, запускается обработка. В это время садоводу уже делать ничего не нужно. К тому же, ночью, препарат не сразу испарится с листьев и будет больше воздействие на растения. Мелко-дисперсное распыление, обеспечивает более равномерное покрытие листьев препаратом. 



### Техническая составляющая

Для управления данным комплексом, используется микроконтроллер Arduino Nano, а именно, управления реле и отслеживания датчиков. 

Также применяется микроконтроллер ESP 8266, для того, чтобы данные с датчиков можно было отправить и принимать команды из телеграмм бота. 

Для удалённого включения насосов, используются 2 релейных модуля на 4 канала. 

![Фото1](https://i.imgur.com/bekZd0e.jpeg)

Для отслеживания скорости ветра был разработан и создан анемометр собственной конструкции, основанный на эффекте Холла.

Корпус и лопасти анемометра распечатаны на 3D принтере. На оси закреплён неодимовый магнит, вращение которого считывает датчик Холла. Arduino принимает импульсы датчика и рассчитывает скорость ветра. 

В нём же, находится датчик BME680 используемый для отслеживания показаний погоды.

Чтобы можно было измерять показания освещённости, используется обычный фоторезистор.

![Фото2](https://i.imgur.com/JI7N8MV.jpeg)

Для работы опрыскивания, на данный момент используется 1 компактный насос на 12В. 

Для питания всей электронной части используется компьютерный блок питания.

А для того, чтобы раствор мелко распылялся и легко оседал на листьях деревьев, используются отдельно распылитель, где расположены 4 форсунки, которые под давлением распрыскивают раствор очень мелко. Мотор и форсунки соединены ПВХ шлангом на 8 мм внутреннего диаметра.

![Фото3](https://i.imgur.com/4SQXuvh.jpeg)

### Программная обеспечение

#### Программное обеспечение микроконтроллера Arduino Nano

Код для Arduino Nano написан на языке C++ в среде Arduino IDE.

Реализован опрос датчика BME680 по шине I2C и фоторезистора через аналоговый вход. Особое внимание уделено измерению скорости ветра: используется механизм прерываний для точного подсчета импульсов датчика Холла, что позволяет разгрузить процессор. Программа принимает команды по последовательному UART порту. Для этого был создан текстовый протокол общения с ESP8266, для переключения реле и отслеживания датчиков.

#### Программное обеспечение сетевого шлюза ESP8266

Модуль с esp8266, выполняет роль связующего звена между устройством и облачным сервером. Для обмена данными между платой управления и сервером, использовал технологию MQTT. Благодаря этому запустить обработку сада, можно везде, важно чтобы был интернет. Для удобства подключения к Wi-Fi использована библиотека WiFiManager. Это позволяет настраивать сеть и привязывать устройство к Telegram ID пользователя без перепрошивки, создавая временную точку доступа. Настройки (ID пользователя и системы) сохраняются в файловую систему в формате JSON. Устройство подписывается на определённый топик для получения команд и публикации данных сенсоров, обеспечивая двустороннюю связь.

#### Программное обеспечение интерфейса управления (Telegram-бот)

Для глобального управления системой разработан бот на языке Python, который выполняет функции сервера. 

Разработка отдельного приложения требует установки программного обеспечения на каждое устройство пользователя, регулярных обновлений, поддержки разных операционных систем, а также сопровождения серверной инфраструктуры для передачи уведомлений и синхронизации данных. Именно поэтому, я и решил первоначально создать телеграмм бота.

Данный бот может управлять множеством систем разных пользователей одновременно без задержек. В коде реализована база данных содержащая настройки пользователей, калибровочные значения, названия деревьев и логов обработок. А также реализована база обработок, содержащая календарь обработок. Бот автоматически определяет текущий месяц и предлагает пользователю актуальные схемы ухода за деревьями.
